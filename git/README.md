## :eye: About
 
_<p align="center">**GIT** is a free and open source distributed version control system designed to handle everything from small to very large projects with speed and efficiency.
<br>
These fully functional local repositories make it easy to work offline or remotely.
<br>
Developers commit their work locally, and then sync their copy of the repository with the copy on the server.
<br>
Nearly every development environment has Git support and Git command line tools implemented on every major operating system.
<br>
<br>
This project has been established by **[Holberton School](https://www.holbertonschool.com/about-holberton)** to enable every student to understand how Git works.</p>_

## :memo: Learning objectives

```diff
+ * What is source code management

+ * What is Git
```

## :computer: Requirements

```diff
+ A README.md file at the root of the repo, containing a description of the repository

+ A README.md file, at the root of the folder of this project (i.e. git), describing what this project is about

- Do not use GitHub’s web UI, but the command line to perform the exercise (except for operations that can not possibly be done any other way than through the web UI). You won’t be able to perform many of the task requirements on the web UI, and you should start getting used to the command line for simple tasks because many complex tasks can only be done via the command line.

+ Your answer files should only contain the command, and nothing else
```


## :package: Installation

**If git is not already installed on your terminal**:

```yaml
$ sudo apt-get update

$ sudo apt-get upgrade

$ sudo apt-get install git
```


## :keyboard: Basic Usage

**You should be able to reproduce and understand these command lines:**

```yaml
$ git clone <repo>
 
$ touch test

$ git add test

$ git commit -m "Initial commit"

$ git push origin main
```
